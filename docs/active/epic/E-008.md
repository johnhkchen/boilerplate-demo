## Intent — the bigger-picture play

Fold today's two-half backstage — a submit-only human form and a separate agent-only retrieval feed — into one passcode-gated dashboard where the human side is no longer write-only. A collaborator unlocks once with the same DEMO_PASSCODE (no second credential, no account), and the authenticated view lets them both submit entries and see all entries together as a checklist: mark each one complete as it is acted on, and delete entries when the project is done or handed off. This matters now because E-003 gave the backstage a door but only an inbox — a stakeholder can push context in but never see what they or others already sent, whether an agent has picked it up, or what is now stale, so the surface silently accretes dead entries and the person who filled it has no way to keep it honest. Making the same shared-passcode surface two-way for management (not just submission) turns the backstage from a drop box into a maintainable shared worklist, exactly the low-tax collaboration P4 promises. Intent only — DecomposeEpic owns the unlock/session, the unified read/list view, the complete and delete operations, and how they compose with the existing agent seam.

## Value to the design

Realizes the management half of the shared-passcode collaboration surface: structured stakeholder context becomes not just submittable but viewable and curatable by the same non-technical collaborators, under one unlock and with zero account or role overhead. That is the substance of P4 (collaboration with no workplace tax) deepened — the backstage stops being a one-way inbox and becomes a shared checklist teammates can keep current. As a permanent it is reusable leverage inherited by every demo the template generates, and it makes the E-007 handoff cleaner by letting an owner clear entries when a project is done or transferred.

## Done looks like

A collaborator, given only the demo URL and the shared passcode, unlocks once and in a single authenticated view submits a new entry, sees every existing entry alongside it as a checklist, marks one complete, and deletes another — with no second credential and no account created; the previously separate submit-only form and agent-only feed no longer exist as two disjoint surfaces, and a coding agent still retrieves the current entries verbatim through the documented seam, reflecting the completes and deletes.

## Context & constraints

Go-and-see: src/** is still empty and the two surfaces this epic consolidates are defined by E-003 stakeholder-backstage (submit form + documented agent-retrieval seam), which is currently OPEN on the board — so E-003 (and the deployed scaffold with server routes and a storage binding behind it, E-001/E-002) is a PREREQUISITE, not hidden scope: there is nothing to fold until the backstage v1 exists. Constraint carried from E-003: the shared passcode is a low-stakes gate, not a server secret — real keys stay out of browser bundles and are never solicited (P3 guardrail, not claimed as an advance); the management operations (complete/delete) must not break or bypass the documented agent-retrieval seam, which keeps reading the same entries. Non-goals respected: per N3/P6 the dashboard is per-project infrastructure in the project's own Cloudflare account, never a central management console; per N5 the one shared passcode stays a shared passcode — this epic adds no registered users, roles, per-user identity, or auth product (that is P5's later revenue path); the storage choice (KV vs D1) is left to decomposition with this concrete read/update/delete need as its justification, not pre-baked here. Out of scope: editing/threading of entries, moderation, file uploads, notifications, audit history, and any multi-user permission model. Right-size (PE-7): unifying the two surfaces under one unlock plus adding list/complete/delete over existing entries fits one ~2-hour block; if decomposition surfaces richer needs (soft-delete/undo, per-entry assignment, agent write-back of completion), those return to the demand board as new signals rather than inflating this card.
