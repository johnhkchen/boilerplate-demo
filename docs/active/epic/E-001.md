_Proposed by Vend's `propose-epic` play._

## Intent — the bigger-picture play

Stand up Demo Runway's first public artifact while the repo is still empty: an Astro site scaffolded in src/, deploying to Cloudflare on every push to main, carrying a polished baseline layout that reads from the back of a room on a projector and works one-handed on a phone. The play is to make the public URL exist before product ideation deepens, so deploy friction, integration errors, and architectural guesses surface against a running site — and so every demo later assembled from this template inherits a foundation that already ships.

## Value to the design

Realizes the template's core promise that a new repository reaches a working public URL before deep ideation (P1). The baseline responsive typography and layout make the same page legible on a projector and usable on a phone — the legibility half of P3; no server-side keys exist yet to protect, and the static bundle gives nothing to leak. Cloudflare static-first hosting keeps idle cost effectively zero and the project independently ownable and transferable (P6). As a permanent, this foundation is the reusable substrate every subsequent epic (observable demo moment, collaboration surface, revenue seams) builds on.

## Done looks like

A teammate given only the repository can open a public Cloudflare URL and read the page comfortably both projected across a room and on a phone; a push to main redeploys it with no manual steps; the initial page loads with no long compute cold start; and the deployed site contains no secrets and no product-specific code — only the foundation.

## Context & constraints

Clears the demand signal 'Deployable demo foundation.' Go-and-see: src/** is empty and the board holds no stories or tickets — this is the founding epic, so E-001 is the next free id. Prerequisites, named as prerequisites rather than hidden scope: a Cloudflare account and authenticated deploy credentials (wrangler locally and/or CI). Non-goals respected: per N1 and N5 the scaffold is Astro static output with minimal dependencies — no React, Tailwind, database, auth, or CMS enters here; the polished baseline uses a small template-owned token layer whose visual direction can evolve through dogfooding rather than depending on a remote kit or prematurely fixed palette. Out of scope and left to later epics: the proven end-to-end demo moment with time-budgeted checks (P2), the no-account stakeholder context surface (P4), user/billing/persistence seams (P5), and fleet tooling (P7). Right-size check (PE-7): scaffold + baseline projector/mobile styling + Cloudflare deploy fits one ~2-hour feature block; it is one epic, not several.
