## Intent — the bigger-picture play

Make a work-in-progress demo behave like a disposable, collaborative venue. Today the template ships a stable public URL; this play makes that URL shareable as capability: an owner starts a session and sends a preview link plus a browser IDE link, the teammate edits an isolated workspace in a clean browser (identity login only — no repo clone, runtime, tunnel, or IDE install) and watches the preview update live, and the owner promotes the strongest session's exact commit to demo.b28.dev reversibly. The handoff becomes a link, not a setup call. (Clears demand.md signal #5.)

## Value to the design

This is the substance of P4 — collaboration with no workplace tax — extended from 'submit structured context' to 'edit the running thing,' realized sovereignly per P6: sessions run as Cloudflare Containers (Sandbox SDK) in the project's own account, no host to babysit, idle cost near zero, nothing central required (honours N3). Promotion as a Workers version alias hardens the stable demo's release story for free, and as a permanent the capability is reusable across every demo the template generates.

## Done looks like

An owner runs one command and receives a preview link and an editor link; a teammate opens the editor in a clean browser, authenticates with an identity login only, edits the isolated session, and sees the preview update live; the owner promotes a healthy session's exact commit to demo.b28.dev and can roll back instantly — no DNS wait, no rebuild — while the public demo and the private session surfaces share no auth bypass.

## Context & constraints

Go-and-see: the stable demo exists on Workers today (promoted through CI); no session runtime exists and src/ is empty. Binding decisions are recorded in docs/knowledge/demo-environments-decisions.md and must be honored: Cloudflare Containers via the Sandbox SDK (not a tunnel+host), promotion as a Workers version alias (not a route swap), a Sessions Worker reverse-proxying branded demo-<slug>/code-<slug> hostnames with Access on the Worker route. PREREQUISITE (not hidden scope): a Phase 0 spike must prove a Sandbox holds astro dev + code-server across idle sleep/wake with HMR over WebSocket through the proxy — the session build is gated on its finding. Guardrails: demo.b28.dev stays public while editor/preview sit behind Access with no shared bypass (P3); secrets are injected at launch, never baked into image or worktree; teardown preserves uncommitted work before destroy; trust boundary is trusted/semi-trusted teammates only. Right-size note (PE-7): this signal is larger than one ~2-hour block — the MVP (Phase 0 spike + stable-domain/promotion + one session + Access/safety + docs) is roughly nine ticket-blocks, several epics' worth of work under one intent; the mana cost states that full MVP envelope rather than inflating scope silently. Out of scope (PRD Phases 3–5): multiple simultaneous sessions, invitations/expiry, garbage collection, reconcile, and any control-plane UI (respects N3).
