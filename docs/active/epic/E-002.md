_Proposed by Vend's `propose-epic` play._

## Intent — the bigger-picture play

Give demos assembled from this template an executable harness that exercises their API boundaries the way the demo does: composable operations that tests and agents can invoke directly, trace output that shows exactly where a call stalls or breaks, and a Playwright flow that runs under an explicit time budget so nothing can spin forever. This matters now because the template currently has no way to distinguish 'the integration works' from 'nobody has looked yet' — without it, the first demo built on this foundation discovers its broken boundaries live, in front of the room.

## Value to the design

Turns integration failure from a live-demo surprise into pre-demo evidence. Advances P2 directly — stalled or broken boundaries become explicit traces and failing checks under a defined time budget instead of an indefinite spinner — and the secret half of P3, by making a server-side key that reaches the browser an observable, failing condition. As a permanent, every future demo assembled from the template inherits this leverage rather than rebuilding it.

## Done looks like

One command run against a demo with a deliberately broken, stalled, or secret-leaking API boundary fails explicitly within its time budget, naming the offending boundary in a trace or Playwright report an agent can read and act on — and the same command passes clean on a healthy demo. No main-flow check can wait indefinitely.

## Context & constraints

Ground truth: src/** is empty. The base template with at least one real API boundary to exercise (the E-001 work now on the board as S-001-01/S-001-02 and their tickets) is a prerequisite, named as such — not hidden scope here. Boundaries respected: per N4 the harness discovers hangs, breakage, and leaks, it does not judge whether the demo is convincing; per N2 the composable operations are seams for boundaries a demo actually has, not an all-provider abstraction layer; per P7 everything runs from the repo with no fleet-service dependency. Out of scope: story/ticket decomposition, fleet uptime checks, and secret-scanning of stakeholder comments or chat (that belongs to the collaboration surface). PE-7 sizing: this is card-sized if the first cut targets one exemplar boundary, one Playwright flow, and a bundle/response secret assertion; if demand later pushes toward a broader operations library or deeper secret scanning, that is a further epic, not growth of this one.
