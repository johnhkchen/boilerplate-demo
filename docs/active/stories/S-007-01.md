**Scope:** A read-and-map pass over the deployable surface — src/session-worker.ts, the backstage/session storage seams (backstage-store.ts, session-lifecycle.ts, session-access.ts), passcode.ts, the check trio (integration-check/leak-check/ops-check) plus the Playwright main flow, and the deploy config (wrangler.jsonc, wrangler.sessions.jsonc, env.d.ts). Produces one inventory artifact under docs/active/work/; touches no runtime code.

**Story acceptance:** An inventory artifact exists that, for each of the seven transfer categories (repo, Cloudflare resources, domain, data, configuration, secrets, checks), names the concrete seam in THIS repo and the specific coupling that binds it to the author — cited by file, binding name, or config key (e.g. the b28.dev custom_domain routes in wrangler.jsonc, the passcode secret), never asserted in the abstract.

**Honest boundary:** This story only MAPS the surface; it transfers nothing and rotates nothing (that is the drill, S-007-02). Free and fixture-free — a static read of the repo and its deploy config, no Cloudflare API calls and no metered actions.

## DAG

```
T-007-01-01  map-transfer-surface  ← T-006-02-02
T-007-01-02  flag-author-couplings  ← T-007-01-01
```

Wave rationale: T-007-01-01 (map the surface) runs alone because the coupling analysis reads its output; T-007-01-02 (flag author couplings) then extends the same artifact. Sequential, one file, no fan-out — the map must settle before couplings can hang off it.

**Out of this slice:** Executing any transfer, standing up a second owner, rotating secrets, writing the runbook, or minting follow-on epics — all downstream. Also excludes inventing new deploy config; it reads what E-001..E-006 established.
