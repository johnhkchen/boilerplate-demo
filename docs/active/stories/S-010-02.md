**Scope:** The Node check slice — src/lib/ops-check.ts (make runBoundaryCheck/formatBoundaryTrace generic over a BoundaryContract's assertShape/verify/name instead of importing assertReceiptShape/verifyReceipt/BOUNDARY_NAME), src/lib/integration-check.ts (derive the pinned boundary field from the contract's name, not typeof BOUNDARY_NAME), and the three runnable edges scripts/ops-check.ts, scripts/leak-check.ts, scripts/integration-check.ts (resolve the boundary URL from `${baseUrl}${contract.path}` and the key env from the contract, dropping the hardcoded /api/receipt).

**Story acceptance:** npm run ops:check, npm run leak:check, and npm run integration:check still pass against the receipt dev server; a unit test drives the now-generic runBoundaryCheck with a FAKE alternate contract instance to pass on a well-formed body, fail on a wrong-shaped body, and fail on a bad signature — with no receipt-specific symbol left in the core; and grep -r '/api/receipt' scripts/ returns nothing.

**Honest boundary:** The flow side (demo-flow.spec.ts, flow-contract.ts, playwright.config.ts) still carries its own literals — pointed at the contract in S-010-03, not here. The full cross-harness swap and npm run verify capstone is S-010-04. Genericity is proven here only at the Node unit + local-server level with a fixture fake contract; FREE, no deploy.
