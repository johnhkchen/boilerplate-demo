**Scope:** The drill harness and the rehearsal run: a reproducible fresh-owner context (author accounts/secrets scrubbed from the runtime path), the per-category transfer attempts against session-worker + backstage/session storage + passcode secrets + the wrangler domain/zone config, and a run of the demo's own checks (integration/leak/ops + Playwright main flow) against the new-owner deployment. Work artifacts land under docs/active/work/T-007-02-*/; drill scripts may be added but product runtime code is not rewritten.

**Story acceptance:** Following the harness, a second-owner context redeploys the Worker and its storage, rotates every secret to new-owner-controlled values, re-points the routes off the author's b28.dev zone, moves stored data, and the demo's own checks run green with the author's accounts and any fleet/central service removed from the runtime path — and every category that could NOT transfer cleanly is recorded with the exact failing seam, not silently skipped.

**Honest boundary:** The drill ATTEMPTS and OBSERVES per PE-7; it does not fully automate any category's transfer (data-migration tooling, secret-rotation tooling, domain re-delegation are follow-on epics minted from the gaps, not built here). Where a real second Cloudflare account is unavailable, the fresh-owner context is a scrubbed local/second-account simulation with that manual live step named explicitly as metered and deferred.

```
T-007-02-01  fresh-owner-drill-harness  ← T-007-01-02
```

Wave rationale: T-007-02-01 (harness) runs alone — it fixes the clean context and the per-category pass/fail signal every attempt reports against. T-007-02-02 (secrets/config) and T-007-02-03 (resources/domain/data) then fan out in parallel: disjoint seams (passcode/env vs. Worker/storage/zone routes) once the harness settles the shared context. T-007-02-04 (checks under new owner) runs last because it can only judge a fully transferred deployment.

**Out of this slice:** The prerequisite deployed demo itself (E-001..E-006); building durable migration/rotation/re-delegation automation (follow-on epics); writing the runbook or gap list (S-007-03). A general all-provider transfer path is barred by N2 — the drill stays on the Cloudflare-first surface.
