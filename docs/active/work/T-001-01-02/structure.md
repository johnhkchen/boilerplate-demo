## Public interface / contract this ticket establishes

The deploy seam that T-001-01-03's CI and any owner depend on:

- `wrangler.jsonc` — the single source of deploy truth. `wrangler deploy` reading
  it uploads `./dist` as an assets-only Worker to `<name>.<subdomain>.workers.dev`
  over HTTPS. Account + token come from the environment, never the file.
- `npm run deploy` — build + deploy, one command (owner happy path; -03 CI target).
- `npm run deploy:dry` — build + `wrangler deploy --dry-run`; offline config +
  bundle validation, no credentials, no upload.

Keeping `wrangler.jsonc` credential-free and the `deploy` script stable is the
seam -03 builds on (it adds only the Actions wrapper + token secret).

## File-by-file specification

### `wrangler.jsonc` (new)

Exactly four meaningful keys, plus schema:

```jsonc
{
  // Cloudflare Workers Static Assets. No `main` → assets-only → the Worker
  // runtime is never invoked for asset requests (no compute cold start).
  // No account_id / tokens here — supplied via env (owner) or CI secret (-03),
  // keeping the repo account-agnostic (P6) and secret-free.
  "$schema": "node_modules/wrangler/config-schema.json",
  "name": "demo-runway",
  "compatibility_date": "2026-07-10",
  "assets": { "directory": "./dist" }
}
```

- `name` — matches the project (`demo-runway`); becomes the `*.workers.dev`
  subdomain label.
- `compatibility_date` — fixed today's date; a deliberate future bump, not
  computed.
- `assets.directory` — `./dist`, Astro's static build output.
- **Absent by design:** `main`, `account_id`, `route(s)`, any `[vars]`/secret/
  binding block. Their absence is the acceptance guarantee (no compute, no
  secrets).

### `package.json` (modified)

- Add to `devDependencies`: `"wrangler": "^4"` (current major; the deploy CLI).
  Still no UI/CSS/DB/auth/CMS dependency — the N1/N5 gate stays green (wrangler is
  tooling).
- Add to `scripts`:
  - `"deploy": "astro build && wrangler deploy"`
  - `"deploy:dry": "astro build && wrangler deploy --dry-run"`
- Leave `dev`/`build`/`preview`/`astro`, `name`, `type`, `version`, `private`
  untouched.

### `package-lock.json` (regenerated)

Regenerated by `npm install` when `wrangler` is added; committed to pin the
resolved wrangler version (+ its transitive tree) for reproducible owner/CI
deploys. Not hand-edited.

### `astro.config.mjs` (modified — comment only)

The existing comment predicts an adapter this ticket decided **not** to add.
Replace the forward-reference so the file tells the truth: static output is
deployed to Cloudflare via **wrangler static assets** (`wrangler.jsonc`), with
**no** SSR adapter; adopting `@astrojs/cloudflare` would be a future idea-driven
SSR ticket. **No code change** — `output: 'static'` and the structure stay
identical; only the comment text changes.

### `.gitignore` (modified)

Append one line: `.wrangler/` — wrangler's local state/cache directory (build
artifacts, deploy metadata). Same rationale as ignoring `dist/`/`.astro/`: keep
the committed tree to source only and deploy diffs clean.

## Build / verify contract (drives the Plan phase)

- `npm install` adds `wrangler`, updates `package-lock.json`, exit 0.
- `npm run build` still emits a fully static `dist/` (no regression from -01):
  `dist/index.html` present, no `_worker.js`/`_routes.json`/`functions/`.
- `npm run deploy:dry` exits 0: wrangler parses `wrangler.jsonc`, resolves the
  `./dist` assets, and builds the upload bundle **without credentials** — proving
  the config is deploy-ready. (If `--dry-run` still demands auth for an assets-
  only deploy, fall back to `wrangler deploy --dry-run --outdir` / config-parse
  validation and record the exact behavior in `progress.md`.)
- Secret grep across tracked files (`wrangler.jsonc`, `package.json`, config)
  for token/key/secret patterns → **no matches** (AC clause 3).
- `.wrangler/` and `dist/` remain untracked after the run.
