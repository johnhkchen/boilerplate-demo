## Context

Makes failure observable on demand: a fault flag with deliberate broken and stalled modes proves the harness names the offender and fails within budget instead of spinning.

_Advances: P2_

## Acceptance Criteria

- [ ] With the fault flag set to 'broken' the ops check exits non-zero reporting the boundary and failure kind; set to 'stalled' it fails at the time budget (not after it), naming the boundary; with the flag unset the identical command passes clean.
